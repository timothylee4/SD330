Timothy Lee's Design Principles

Key Principles

  • Visibility
  • Consistency
  • Feedback
  • Constraints
  • Affordances
  • Accessibility

Complexity Considerations

  • Use icons with obvious meanings
  • Keep it simple, stupid (K.I.S.S.)
  • Don’t make users think (Hick’s Law)
  • Make our side more complicated to give users an easier time

Cultural Considerations

  • Keep in mind cultures that read right to left vs. left to right
  • Choose colors, hand gestures, and words carefully depending on the culture
  • Be aware of language differences
  • Make sure to include the ideas of people from different cultural backgrounds
  • Cater to males & females - have both genders help with the design
  • Be aware of cultural norms / etiquette
  • Be mindful of cultural holidays

Human Centered Design Principles

  • Easy to use!
  • Allow users to learn by doing
  • Use affordances (hints that a certain action can be done) and signifiers (things to lead users to affordances)
  • Provide discoverability
  • Utilize a conceptual model
  • Provide feedback (immediate, reactive, clear, customizable, haptic)
  • Have good mapping
  • Accommodate users (users may make mistakes, but they are never wrong)
  • Make sure the user doesn’t worry about any underlying things (WYSIWYG)
  • Set up contraints for the user

Conceptional Principles

  • Having a certain level of complexity is ok (Tesler’s Law)
  • Make site work on the ideas that users are already familiar with (Jakob’s Law)

Interface Principles

  • Make important information visually salient
  • Mobile Devices
    • Make design accessible for both left and right handed people + big or small hands
    Use familiar navigation
    Use familiar navigation
  • Make it aesthetic to a certain degree - don’t only focus on the aesthetics
  • Don’t overwhelm the user with clutter
  • Make interface intuitive - similar to what users already use

Emotional / Accessibility Principles

  • Have empathy for users
  • Make sure it is accessible for as many users as possible, including:
    • Sensory impaired
    Physically impaired
    Cognitively impaired
  • Feature inclusive design of technology & assistive design
  • Make sure to include accessibility by making final product:
    • Perceivable - Information and UI must be presented in a way users can take in
    Operable - US components and navigation must be able to use the interface
    Understandable - Users must be able to comprehend information and operation of user interfaces
    Robust - Content should be interpreted reliably by a wide variety of users

Functional Design Principles

  • Do everything to prevent errors - the best error message is no error message
  • Make sure all use cases are fleshed out
  • Make sure to keep errors in mind - understand the cause and try to minimize them
  • Perform sensibility checks - do the actions make sense?
  • Make it possible to reverse actions
